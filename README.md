Sistema Web de Gestión de Turnos – Veterinaria Huellas
- Aplicación web desarrollada como proyecto académico en el curso de Ingeniería de Software, orientada a la gestión de turnos para una veterinaria.
- El sistema permite a los clientes reservar turnos para distintos servicios y al administrador gestionar las reservas de manera eficiente.

---

Tecnologías utilizadas

- HTML5
- CSS3
- JavaScript
- Jest
- LocalStorage

---

Funcionalidades principales

Usuario (cliente)
- Reserva de turnos seleccionando servicio y fecha
- Visualización de horarios disponibles
- Ingreso de datos del cliente y mascota
- Validaciones de formularios

Administrador
- Login mediante credenciales
- Visualización de listado de turnos reservados
- Acceso a información completa de reservas

Generales
- Navegación fluida entre secciones
- Diseño responsive (adaptable a mobile y desktop)
- Secciones informativas:
  - Servicios
  - Profesionales
  - Galería
  - Contacto con mapa

---

Conceptos aplicados

Ingeniería de Software
- Relevamiento de requerimientos
- Historias de usuario (User Stories)
- Priorización mediante método **MoSCoW**
- Definición de criterios de aceptación
- Planificación de sprint

Desarrollo
- Implementación en JavaScript sin frameworks
- Separación de estructura, estilo y lógica
- Manejo de estados con LocalStorage

Testing
- Testing unitario con **Jest**
- Validación de lógica crítica (login, reservas)

Trabajo en equipo
- Organización mediante **Trello**
- Distribución de tareas por funcionalidades
- Seguimiento del progreso del sprint

---

Arquitectura del proyecto
- `index.html` → estructura de la aplicación (SPA)
- `styles.css` → estilos
- `app.js` → lógica principal
- `app.test.js` → pruebas unitarias
- `/img` → recursos visuales

---

Metodología de trabajo

Se utilizó una metodología ágil basada en sprints:

- Planificación mediante backlog
- Estimación con **Planning Poker**
- Definición de **Definition of Done (DoD)**
- Seguimiento de tareas en Trello

---

Cómo ejecutar

1. Clonar o descargar el repositorio
2. Abrir `index.html` en un navegador

---

Objetivo del proyecto
Desarrollar una aplicación web completa aplicando el ciclo de vida del software, desde el relevamiento de requerimientos hasta la implementación y testing, simulando un entorno de trabajo real.
